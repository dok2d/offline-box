# Calibre-web

Веб-интерфейс для управления электронной библиотекой Calibre. Позволяет просматривать, искать и скачивать электронные книги через браузер. Поддерживает загрузку готовой библиотеки из архива или создание пустой библиотеки при первом запуске.

## Переменные

| Переменная | Описание | Значение по умолчанию |
|-----------|----------|----------------------|
| `enable_calibre_web` | Включить/выключить сервис | `true` |
| `calibre_web_port` | Порт веб-интерфейса | `8008` |
| `calibre_web_data_dir` | Каталог для хранения данных | `/opt/calibre-web/data` |
| `calibre_web_config_dir` | Каталог конфигурации | `/opt/calibre-web/config` |
| `calibre_web_library_dir` | Каталог библиотеки Calibre | `/opt/calibre-web/data/library` |
| `calibre_web_library_url` | URL zip-архива с библиотекой Calibre (должен содержать `metadata.db`). Если пустой, создаётся пустая библиотека | `""` (пустая строка) |

## Порт

Сервис слушает на `127.0.0.1:8008`. Доступен через nginx по пути `/calibre/`.

## Данные

- **Данные**: `{{ opt_base }}/calibre-web/data`
  - `library/` -- каталог библиотеки с файлами книг и `metadata.db`
- **Конфигурация**: `{{ opt_base }}/calibre-web/config` -- Containerfile, `app.db` (база настроек приложения)

## Инициализация

При первом развёртывании выполняются следующие действия:

1. Создание каталогов: данные, конфигурация и библиотека
2. Если задан `calibre_web_library_url`:
   - Скачивание zip-архива библиотеки
   - Распаковка архива в каталог библиотеки
3. Если URL не задан -- создание пустой базы `metadata.db` с минимальной схемой через Python/SQLite
4. Сборка контейнерного образа `localhost/calibre-web` из Containerfile
5. Создание и запуск systemd user unit
6. Ожидание готовности Calibre-web
7. Автоматическая настройка пути к библиотеке в `app.db`
8. Перезапуск сервиса для применения настроек
9. Настройка nginx reverse proxy

Учётные данные по умолчанию для веб-интерфейса Calibre-web: `admin` / `admin123` (стандартные для Calibre-web).

## Зависимости

- Роли: `base`, `podman`, `nginx`
