# Gitea

Легковесный Git-сервер с веб-интерфейсом для хостинга репозиториев. Поддерживает Git LFS, систему задач (issues), pull-запросы и SSH-доступ. Использует SQLite в качестве базы данных по умолчанию.

## Переменные

| Переменная | Описание | Значение по умолчанию |
|-----------|----------|----------------------|
| `enable_gitea` | Включить/выключить сервис | `true` |
| `gitea_port` | Порт веб-интерфейса | `8006` |
| `gitea_data_dir` | Каталог для хранения данных | `/opt/gitea/data` |
| `gitea_config_dir` | Каталог конфигурации | `/opt/gitea/config` |
| `gitea_db_type` | Тип базы данных | `sqlite3` |
| `gitea_secret_key` | Секретный ключ для подписи токенов | (из passwords.yml) |
| `gitea_lfs_enabled` | Включить поддержку Git LFS | `true` |
| `gitea_ssh_port` | Порт SSH для Git-операций | `2222` |
| `gitea_admin_user` | Имя пользователя администратора | `admin` |
| `gitea_admin_password` | Пароль администратора | (из passwords.yml) |
| `gitea_admin_email` | Email администратора | `admin@localhost` |

## Порт

Сервис слушает на `127.0.0.1:8006`. Доступен через nginx по пути `/gitea/`.
SSH-порт для Git-операций: `2222`.

## Данные

- **Данные**: `{{ opt_base }}/gitea/data` -- репозитории, LFS-объекты, база данных SQLite
- **Конфигурация**: `{{ opt_base }}/gitea/config` -- `app.ini`, Containerfile

## Инициализация

При первом развёртывании выполняются следующие действия:

1. Создание каталогов для данных и конфигурации
2. Генерация конфигурации `app.ini` из шаблона
3. Сборка контейнерного образа `localhost/gitea` из Containerfile
4. Создание и запуск systemd user unit
5. Ожидание готовности Gitea
6. Создание пользователя-администратора через `gitea admin user create` (при повторном запуске пропускается, если пользователь уже существует)
7. Настройка nginx reverse proxy

## Зависимости

- Базовый образ: `docker.io/library/debian:13-slim`
- Роли: `base`, `podman`, `nginx`
