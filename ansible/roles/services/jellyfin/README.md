# Jellyfin

Медиасервер для стриминга видео, аудио и фотографий. Предоставляет веб-интерфейс для управления медиатекой и просмотра контента с любого устройства в локальной сети. По умолчанию настроен на русский язык.

## Переменные

| Переменная | Описание | Значение по умолчанию |
|-----------|----------|----------------------|
| `enable_jellyfin` | Включить/выключить сервис | `true` |
| `jellyfin_port` | Порт веб-интерфейса | `8007` |
| `jellyfin_data_dir` | Каталог для хранения данных | `/opt/jellyfin/data` |
| `jellyfin_config_dir` | Каталог конфигурации | `/opt/jellyfin/config` |
| `jellyfin_media_dir` | Каталог для медиафайлов | `/opt/jellyfin/data/media` |
| `jellyfin_cache_dir` | Каталог кеша | `/opt/jellyfin/data/cache` |
| `jellyfin_admin_user` | Имя пользователя администратора | `admin` |
| `jellyfin_admin_password` | Пароль администратора | (из passwords.yml) |
| `jellyfin_language` | Язык интерфейса и метаданных | `ru` |
| `jellyfin_country` | Страна для метаданных | `RU` |

## Порт

Сервис слушает на `127.0.0.1:8007`. Доступен через nginx по пути `/jellyfin/`.

## Данные

- **Данные**: `{{ opt_base }}/jellyfin/data`
  - `media/` -- каталог для медиафайлов (видео, аудио, фото)
  - `cache/` -- кеш метаданных и транскодирования
- **Конфигурация**: `{{ opt_base }}/jellyfin/config` -- Containerfile, внутренняя конфигурация Jellyfin

## Инициализация

При первом развёртывании выполняются следующие действия:

1. Создание каталогов: данные, конфигурация, медиа и кеш
2. Сборка контейнерного образа `localhost/jellyfin` из Containerfile
3. Создание и запуск systemd user unit
4. Ожидание готовности Jellyfin (healthcheck)
5. Прохождение мастера начальной настройки:
   - Установка языка интерфейса и региона
   - Создание администратора
   - Настройка медиабиблиотеки
   - Завершение мастера настройки
6. Настройка nginx reverse proxy

Для добавления медиаконтента скопируйте файлы в каталог `jellyfin_media_dir`.
