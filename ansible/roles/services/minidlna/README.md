# MiniDLNA

Лёгкий DLNA/UPnP медиасервер для трансляции видео, аудио и изображений на устройства в локальной сети (Smart TV, медиаплееры, игровые консоли). Контент автоматически индексируется и становится доступен по протоколу DLNA без дополнительной настройки на клиентских устройствах.

## Переменные

| Переменная | Описание | Значение по умолчанию |
|-----------|----------|----------------------|
| `enable_minidlna` | Включить/выключить сервис | `true` |
| `minidlna_port` | Порт веб-интерфейса статуса | `8017` |
| `minidlna_data_dir` | Каталог для хранения данных | `/opt/minidlna/data` |
| `minidlna_config_dir` | Каталог конфигурации | `/opt/minidlna/config` |
| `minidlna_media_dir` | Каталог для медиафайлов | `/opt/minidlna/data/media` |
| `minidlna_db_dir` | Каталог базы данных индекса | `/opt/minidlna/data/db` |
| `minidlna_friendly_name` | Имя сервера в сети DLNA | `Offline Box DLNA` |

## Порт

Веб-интерфейс статуса слушает на `127.0.0.1:8017`. Доступен через nginx по пути `/minidlna/`.
Для обнаружения по DLNA/UPnP контейнер запускается с `--network=host`.

## Данные

- **Данные**: `{{ opt_base }}/minidlna/data`
  - `media/` — каталог для медиафайлов (видео, аудио, изображения)
  - `db/` — база данных индекса медиафайлов
- **Конфигурация**: `{{ opt_base }}/minidlna/config` — Containerfile, minidlna.conf

## Инициализация

При первом развёртывании выполняются следующие действия:

1. Создание каталогов: данные, конфигурация, медиа и база данных
2. Сборка контейнерного образа `localhost/minidlna` из Containerfile
3. Развёртывание конфигурации minidlna.conf
4. Создание и запуск systemd user unit
5. Ожидание готовности MiniDLNA (проверка веб-интерфейса статуса)
6. Настройка nginx reverse proxy для страницы статуса

Для добавления медиаконтента скопируйте файлы в каталог `minidlna_media_dir`.

## Зависимости
