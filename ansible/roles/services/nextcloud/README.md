# Nextcloud

Облачное хранилище файлов с возможностями совместной работы. Позволяет хранить, синхронизировать и делиться файлами, а также работать с документами, календарями и контактами. Использует SQLite в качестве базы данных по умолчанию.

## Переменные

| Переменная | Описание | Значение по умолчанию |
|-----------|----------|----------------------|
| `enable_nextcloud` | Включить/выключить сервис | `true` |
| `nextcloud_port` | Порт, на котором слушает сервис | `8005` |
| `nextcloud_data_dir` | Каталог для хранения данных | `/opt/nextcloud/data` |
| `nextcloud_config_dir` | Каталог конфигурации | `/opt/nextcloud/config` |
| `nextcloud_admin_user` | Имя пользователя администратора | `admin` |
| `nextcloud_admin_password` | Пароль администратора | `changeme` |
| `nextcloud_db_type` | Тип базы данных | `sqlite` |
| `nextcloud_upload_max_size` | Максимальный размер загружаемого файла | `10G` |
| `nextcloud_php_memory_limit` | Лимит памяти PHP | `512M` |

## Порт

Сервис слушает на `127.0.0.1:8005`. Доступен через nginx по пути `/nextcloud/`.

## Данные

- **Данные**: `{{ opt_base }}/nextcloud/data` -- пользовательские файлы, база данных SQLite
- **Конфигурация**: `{{ opt_base }}/nextcloud/config` -- Containerfile, `autoconfig.php`

## Инициализация

При первом развёртывании выполняются следующие действия:

1. Создание каталогов для данных и конфигурации
2. Генерация файла `autoconfig.php` из шаблона (автоматическая настройка при первом запуске)
3. Сборка контейнерного образа `localhost/nextcloud` из Containerfile
4. Создание и запуск systemd user unit
5. Ожидание готовности Nextcloud
6. Запуск автоматической установки (создание администратора и БД при первом обращении)
7. Настройка nginx reverse proxy

## Зависимости

- Базовый образ: `docker.io/library/debian:13-slim`
- Роли: `base`, `podman`, `nginx`
