# Syncthing

Инструмент для непрерывной децентрализованной синхронизации файлов между устройствами. Работает по принципу peer-to-peer без центрального сервера. Поддерживает шифрование, версионирование и автоматическое обнаружение устройств в локальной сети.

## Переменные

| Переменная | Описание | Значение по умолчанию |
|-----------|----------|----------------------|
| `enable_syncthing` | Включить/выключить сервис | `true` |
| `syncthing_port` | Порт веб-интерфейса (GUI) | `8011` |
| `syncthing_data_dir` | Каталог для хранения данных | `/opt/syncthing/data` |
| `syncthing_config_dir` | Каталог конфигурации | `/opt/syncthing/config` |
| `syncthing_sync_port` | Порт для синхронизации между устройствами | `22000` |
| `syncthing_discovery_port` | Порт для обнаружения устройств (UDP) | `21027` |
| `syncthing_gui_user` | Имя пользователя для веб-интерфейса | `admin` |
| `syncthing_gui_password` | Пароль для веб-интерфейса | `changeme` |

## Порт

Сервис слушает на `127.0.0.1:8011`. Доступен через nginx по пути `/syncthing/`.
Порт синхронизации: `22000` (TCP).
Порт обнаружения: `21027` (UDP).

## Данные

- **Данные**: `{{ opt_base }}/syncthing/data` -- синхронизируемые файлы
- **Конфигурация**: `{{ opt_base }}/syncthing/config` -- Containerfile, `config.xml` (создаётся при первом запуске)

## Инициализация

При первом развёртывании выполняются следующие действия:

1. Создание каталогов для данных и конфигурации
2. Сборка контейнерного образа `localhost/syncthing` из Containerfile
3. Создание и запуск systemd user unit
4. Ожидание готовности Syncthing (healthcheck)
5. Получение API-ключа из `config.xml` контейнера
6. Установка учётных данных для веб-интерфейса (логин/пароль) через REST API
7. Создание общей папки по умолчанию (`Default`, путь `/data/Sync`)
8. Настройка nginx reverse proxy

## Зависимости

- Роли: `base`, `podman`, `nginx`
